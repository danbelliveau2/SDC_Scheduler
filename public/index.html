Estimate → Schedule Rules

Logic the auto-generator uses when you click + Open project → Analyze on an SDC estimate xlsx. These are read-only for now; tell me which ones need to become editable.

  1. Default headcount = 1 per discipline (Mech Eng / Controls Eng / General Eng / Build / Wire). Auto-bumps only when the minimum critical path overruns delivery.
  2. Backlog = 2 weeks (editable) — kickoff / paperwork / ramp-up time between Receipt of PO and the actual start of engineering. Rendered as a 0% allocation task above Mechanical Design.
  3. Allocation tiers: 25%, 50%, or 90% — never anything else. Tasks snap to the nearest tier, erring toward more time when between tiers.
  4. Mech 1 Release = end of mechanical engineering (FF the latest mech-eng task with no lag).
  5. Build start = Mech 1 Release + 6 weeks (procurement lag for mechanical parts).
  6. Wire ends ~1 week after Build (FF + 1 week for small jobs, more for larger).
  7. Wire bucket split 1:3 — Build and Wire Panel gets 25% of total wire hours, Wire Machine gets 75%.
  8. Section 40 testing = 2 controls fixed (lead debugger + secondary, both CE Placeholder tasks).
  9. Configure Machine = 5% of section-40 engineering hours (half a week small / 1 week large).
  10. Test/Debug 1 + 2 share remaining 95% running in parallel.
  11. Shop Debug duration = engineering debug duration (parallel; headcount auto-scales with hours).
  12. FAT = Test/Debug Engineer 1 FF − 1 week (FAT happens 1 week before main test/debug ends — that final week is finishing-up work).
  13. Section 50 Teardown + Install: 1 week each (fixed). More hours = more people, not more days.
  14. Mech Eng STRETCHES to absorb schedule slack — allocation drops from 90 → 50 → 25 as the delivery window grows past the minimum critical path, pushing Mech 1 Release as late as possible.
  15. Duplicate template tasks collapse when headcount = 1: ME 2 disappears when 1 mech engineer; Builder 2 disappears when 1 builder. One person, full time.
  16. Hours per task come from the SUMMARY FOR RELEASE tab, broken out by column: Mech Eng → Mechanical Design, ce_design → Controls Design + Drawings, ce_software → Controls Software, gen_hmi → HMI Programming, gen_robot → Robot Programming, gen_vision → Vision Programming, etc. Each template task gets the estimate hours for its specific column.
